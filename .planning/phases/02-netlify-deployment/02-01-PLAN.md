  <action>
Guide the user through connecting their GitHub repository to Netlify via the web UI. This matches Kevin Powell's tutorial approach of using the Netlify dashboard (not CLI).

**Steps to guide the user through:**

1. **Go to Netlify Dashboard:**
   - Open https://app.netlify.com
   - If not logged in, sign up / log in (GitHub login recommended for easier repo access)

2. **Import the repository:**
   - Click "Add new site" > "Import an existing project"
   - Select "GitHub" as the Git provider
   - Authorize Netlify to access GitHub if prompted
   - Find and select the `codementor-jamstack-blog` repository

3. **Configure build settings:**
   - **Branch to deploy:** `main`
   - **Build command:** `npm run build`
   - **Publish directory:** `public`
   - Explain WHY each setting: build command runs Eleventy to generate static HTML from templates; publish directory matches `output: "public"` in eleventy.config.js; branch matches the repository's default branch

4. **Deploy the site:**
   - Click "Deploy site" (or equivalent button)
   - Wait for the first build to complete (usually 1-2 minutes)

**Verification checkpoint — you should see:**
- Build log shows "Eleventy" output with pages processed
- Deploy status shows "Published" (green)
- A `.netlify.app` URL is generated (e.g., `random-name-12345.netlify.app`)

**Troubleshooting if build fails:**
- "Command not found" error: verify build command is exactly `npm run build`
- "No such directory" error: verify publish directory is `public` (not `_site`)
- Dependency errors: check build log for missing packages — Eleventy and luxon should install from package.json
