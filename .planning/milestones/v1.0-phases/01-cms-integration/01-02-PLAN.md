  <name>Task 1: Add passthrough copy for admin folder (CMS-05)</name>
  <files>eleventy.config.js</files>
  <action>
**Walkthrough delivery:** One step at a time, explain what AND why.

**Step 1 — Explain passthrough copy concept:** Eleventy processes template files (Nunjucks, Markdown) and transforms them into HTML. But some files should be copied as-is without processing — CSS files, images, and our CMS admin files. That's what `addPassthroughCopy` does: it tells Eleventy "just copy these files to the output directory unchanged."

The project already uses passthrough copy for:
- `./src/style.css` → `public/style.css`
- `./src/assets` → `public/assets/`

We need to add the same for admin files:
- `./src/admin` → `public/admin/`

**Step 2 — Instruct the edit:** Tell the user to open `eleventy.config.js` and add one line after the existing `addPassthroughCopy` calls:

```javascript
eleventyConfig.addPassthroughCopy('./src/admin')
```

The relevant section should look like:
```javascript
eleventyConfig.addPassthroughCopy('./src/style.css')
eleventyConfig.addPassthroughCopy('./src/assets')
eleventyConfig.addPassthroughCopy('./src/admin')
```

Explain: Without this line, Eleventy ignores the admin folder during build. The CMS files would exist in `src/admin/` but never appear in `public/admin/`, so visiting `/admin` would return a 404 error.

**Step 3 — Verify:** After user confirms the edit, read back `eleventy.config.js` to verify:
- The new line is present
- It's inside the `module.exports` function
- Path is `'./src/admin'` (with the `./` prefix matching the existing pattern)
If errors found, guide user to fix them.
