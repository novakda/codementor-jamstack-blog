**Change 1: Add Identity widget to base.njk `<head>`**

Open `src/_includes/base.njk` and add one line inside the `<head>` tag, after the stylesheet link:

```html
<script src="https://identity.netlify.com/v1/netlify-identity-widget.js"></script>
```

Explain: This loads the Netlify Identity widget on every page of your site. It needs to be site-wide (not just /admin/) because when Netlify sends invite emails, the link goes to your root domain (e.g., `yoursite.netlify.app/#invite_token=xxx`). The widget on the homepage catches that token and handles authentication.

**Change 2: Add redirect script to base.njk before `</body>`**

Still in `src/_includes/base.njk`, add this script block just before the closing `</body>` tag (after the footer include):

```html
<script>
  if (window.netlifyIdentity) {
    window.netlifyIdentity.on("init", user => {
      if (!user) {
        window.netlifyIdentity.on("login", () => {
          document.location.href = "/admin/";
        });
      }
    });
  }
</script>
```

Explain each part:
- `if (window.netlifyIdentity)` — Checks the Identity widget loaded successfully (defensive coding)
- `.on("init", user => {...})` — Runs when the widget initializes; `user` is the currently logged-in user or `null`
- `if (!user)` — Only sets up the redirect if nobody is logged in (prevents redirect loops for already-authenticated users)
- `.on("login", () => {...})` — After a successful login, redirects the browser to the /admin/ CMS page

The complete `base.njk` should look like:
```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{{ title }}</title>
    <link rel="stylesheet" href="/style.css">
    <script src="https://identity.netlify.com/v1/netlify-identity-widget.js"></script>
  </head>
  <body>
    {% include 'header.njk' %}
    <main>
    {{ content | safe }}
    </main>
    {% include 'footer.njk' %}
    <script>
      if (window.netlifyIdentity) {
        window.netlifyIdentity.on("init", user => {
          if (!user) {
            window.netlifyIdentity.on("login", () => {
              document.location.href = "/admin/";
            });
          }
        });
      }
    </script>
  </body>
</html>
```

- The file has the redirect script before `</body>`

**Change 4: Switch CMS backend to git-gateway**

Open `src/admin/config.yml` and change the backend section from:
```yaml
backend:
  name: test-repo
```
to:
```yaml
backend:
  name: git-gateway
  branch: main
```

Explain:
- `name: git-gateway` — Tells Decap CMS to authenticate through Netlify's Git Gateway service instead of the local test-repo. Git Gateway proxies your commits to GitHub using Netlify's credentials, so CMS users don't need GitHub accounts.
- `branch: main` — Explicitly targets the main branch. This is important because Git Gateway defaults to `master`, but your repository uses `main`. Without this line, commits would fail silently or go to the wrong branch.

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Enable Netlify Identity and Git Gateway</name>
  <action>
Guide the user through enabling authentication services in the Netlify dashboard.

**Step 1: Enable Netlify Identity**
1. Go to your Netlify dashboard > select your site
2. Navigate to: **Site configuration** (left sidebar) > **Identity**
3. Click **"Enable Identity"**
4. Under **Registration**, select **"Invite only"** (this prevents random signups — you control who can access the CMS)

Explain: Netlify Identity is a built-in authentication service. Enabling it creates user management endpoints on your site (at `/.netlify/identity/`). The "Invite only" setting means only people you explicitly invite can create accounts — important for a CMS where you don't want strangers editing your content.

**Verification checkpoint — you should see:**
- Identity page shows "Identity is enabled"
- Registration preferences shows "Invite only"

**Step 2: Enable Git Gateway**
1. Still on the Identity page, scroll down to **Services**
2. Find **Git Gateway** and click **"Enable Git Gateway"**
3. If prompted, authorize with GitHub (Netlify needs permission to create commits on your behalf)

Explain: Git Gateway is the bridge between Netlify Identity users and your GitHub repository. When a CMS user saves a post, Git Gateway takes their authentication token (from Identity), validates it, and makes the actual GitHub API call to create a commit. This way, CMS users never need GitHub accounts.

**Verification checkpoint — you should see:**
- Git Gateway shows "Enabled" status under Services
- GitHub authorization completed (if prompted)

**Step 3: Invite yourself as a user**
1. Navigate to: **Site configuration** > **Identity** > **Users** tab (or **Identity** in the main nav)
2. Click **"Invite users"**
3. Enter your email address and click "Send"

Explain: Since registration is invite-only, you need to invite yourself before you can log in. Netlify will send an email with an "Accept the invite" link. The link contains an invite token that the Identity widget (which you added to base.njk) will detect and process.

**Verification checkpoint — you should see:**
- Your email appears in the Users list with "Invited" status
- Check your email inbox (and spam folder) for an invite from Netlify
